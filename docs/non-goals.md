# ReelsMaker — Non-Goals (하지 않을 것들)

> 최종 갱신: 2026-03-24

이 문서는 ReelsMaker v1에서 **의도적으로 구현하지 않는 것들**을 명시한다.
비용·복잡도 대비 가치가 낮거나, 제품의 핵심 목표와 맞지 않아 제외한다.

---

## 1. 로그인 / 인증

| 항목 | 설명 |
|------|------|
| 제외 대상 | 회원가입, 로그인, 세션 관리, JWT, OAuth |
| 이유 | 1인 내부 도구. 사용자가 1명이므로 인증 자체가 불필요 |
| 보안 대안 | 서비스를 localhost 또는 VPN 뒤에서만 실행 |
| 확장 경로 | 필요 시 FastAPI 미들웨어에 API Key 인증 추가 (30분 작업) |

---

## 2. 팀 협업

| 항목 | 설명 |
|------|------|
| 제외 대상 | 멀티유저, 역할 분리 (편집자/리뷰어/관리자), 공유 링크, 코멘트 |
| 이유 | 동시 편집·충돌 해결·알림 시스템 등은 핵심 파이프라인 대비 복잡도가 매우 높음 |
| 확장 경로 | DB에 owner_id 추가 + auth 미들웨어 + 권한 체크로 확장 가능 |

---

## 3. 결제 / 플랜 / 사용량 제한

| 항목 | 설명 |
|------|------|
| 제외 대상 | 구독 플랜, 결제 연동 (Stripe 등), 크레딧 시스템, 사용량 할당량 |
| 이유 | 내부 도구이므로 과금 대상이 없음. AI API 비용은 개발자가 직접 관리 |
| 비용 추적 | ProviderRun.cost_estimate로 API 호출별 비용을 기록하여 참고용으로 제공 |

---

## 4. 퍼블릭 SaaS 운영 기능

| 항목 | 설명 |
|------|------|
| 제외 대상 | 커스텀 도메인, SSL 인증서 관리, CDN 설정, 로드밸런싱, 오토스케일링, 모니터링 대시보드 |
| 이유 | 로컬/개인 서버에서만 실행하므로 프로덕션 인프라 불필요 |
| 현재 인프라 | Docker Compose 단일 머신. PostgreSQL, Redis, MinIO 모두 로컬 컨테이너 |

---

## 5. 고급 권한 관리

| 항목 | 설명 |
|------|------|
| 제외 대상 | RBAC, 프로젝트별 접근 제어, API 키 관리 UI, 감사 로그 |
| 이유 | 사용자가 1명이므로 "누가 무엇을 했는지" 추적할 필요 없음 |
| 대안 | 모든 변경은 created_at/updated_at으로 시간 추적만 수행 |

## 6. 외부 배포 자동화

| 항목 | 설명 |
|------|------|
| 제외 대상 | CI/CD 파이프라인, 자동 배포 스크립트, Kubernetes, Terraform |
| 이유 | 단일 머신에서 docker compose up으로 충분. 배포 자동화는 오버엔지니어링 |
| 현재 배포 | `docker compose up -d` 한 줄 |

## 7. 실시간 기능

| 항목 | 설명 |
|------|------|
| 제외 대상 | WebSocket 실시간 업데이트, 생성 과정 라이브 스트리밍, 실시간 공동 편집 |
| 이유 | 단일 사용자이므로 폴링으로 충분. WebSocket 인프라 복잡도 불필요 |
| 현재 방식 | 프론트엔드가 Job 상태를 주기적으로 폴링 (2–5초 간격) |
| 확장 경로 | 필요 시 SSE(Server-Sent Events)로 가볍게 전환 가능 |

## 8. 고급 영상 편집

| 항목 | 설명 |
|------|------|
| 제외 대상 | 구간 트리밍, 키프레임 애니메이션, 이펙트/필터, 오디오 믹싱, 멀티트랙 타임라인 에디터 |
| 이유 | 전문 NLE(Non-Linear Editor)를 만드는 것이 아님. 핵심은 AI 생성 파이프라인 |
| 현재 범위 | Scene/Shot/FrameSpec 순서 기반 자동 타임라인 + 트랜지션(cut/fade/dissolve) |
| 대안 | 세밀한 편집이 필요하면 렌더된 영상을 DaVinci Resolve 등 전문 도구에서 후처리 |

|------|------|
| 제외 대상 | UI 다국어 지원, 자동 번역, 다국어 자막 |

## 경계 판단 기준

새 기능 요청이 있을 때, 아래 질문으로 non-goal 여부를 판단한다:

1. **이 기능 없이도 고품질 영상을 제작할 수 있는가?** → Yes면 non-goal 후보
2. **1인 사용자에게 의미 있는 가치를 주는가?** → No면 non-goal
3. **핵심 파이프라인의 견고함에 기여하는가?** → No면 non-goal
4. **구현 복잡도 대비 사용 빈도가 합리적인가?** → No면 non-goal

위 질문 중 2개 이상에 해당하면, 해당 기능은 이 문서에 추가하고 구현하지 않는다.
