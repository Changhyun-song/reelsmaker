# ReelsMaker — Product Requirements Document

> 최종 갱신: 2026-03-24

## 1. 제품 목표

ReelsMaker는 **1인 개발자가 고품질 AI 영상을 체계적으로 제작**하기 위한 로컬 전용 내부 도구다.

| 목표 | 설명 |
|------|------|
| 고품질 출력 | 프레임 단위의 세밀한 제어로 유튜브 쇼츠/릴스 수준 영상 품질 확보 |
| 구조화된 파이프라인 | 주제 → 대본 → 구조화 → 생성 → 합성 → 렌더까지 끊김 없는 워크플로우 |
| 부분 재생성 | 불만족/실패 구간만 재생성. 전체 재작업 방지 |
| 비동기 처리 | 이미지·비디오·TTS 등 무거운 작업은 큐에서 처리. UI는 항상 반응 |
| 산출물 추적 | 모든 생성물의 버전·파라미터·프로바이더 응답을 기록하여 디버깅과 비용 추적 가능 |

이 제품은 **공개 서비스가 아니다**. 개발자 본인만 사용하는 내부 툴이며,
인증·결제·팀 기능을 의도적으로 배제하여 생성 파이프라인의 견고함에 집중한다.

## 2. 핵심 사용자 플로우

```
┌─ 1. 프로젝트 생성 ─────────────────────────────────────────┐
│    제목, 스타일 프리셋, 해상도/fps, 캐릭터 등록              │
└──────────────────────────┬──────────────────────────────────┘
                           ▼
┌─ 2. 대본 작성 ──────────────────────────────────────────────┐
│    주제/키워드 입력 → AI가 내레이션 초안 생성                 │
│    사용자가 수동 편집/승인 (ScriptVersion으로 버전 관리)       │
└──────────────────────────┬──────────────────────────────────┘
                           ▼
┌─ 3. 구조화 ─────────────────────────────────────────────────┐
│    대본 → Scene → Shot → FrameSpec 자동 분해                 │
│    각 레벨 수동 편집 (순서 변경, 추가, 삭제, 프롬프트 조정)    │
└──────────────────────────┬──────────────────────────────────┘
                           ▼
┌─ 4. 에셋 생성 ──────────────────────────────────────────────┐
│    "생성 시작" → 이미지 → 비디오 → TTS → 자막 비동기 생성     │
│    Frame/Shot 단위 프리뷰 → 불만족 시 부분 재생성             │
└──────────────────────────┬──────────────────────────────────┘
                           ▼
┌─ 5. 타임라인 합성 + 렌더 ───────────────────────────────────┐
│    Timeline 자동 구성 → 전체 프리뷰                          │
│    최종 렌더(FFmpeg) → mp4 다운로드                          │
└─────────────────────────────────────────────────────────────┘
```

## 3. 주요 기능

### 3.1 대본 관리

- 주제/키워드 기반 AI 대본 생성 (GPT-4o)
- 수동 대본 입력 및 편집
- ScriptVersion 기반 버전 관리: 수정할 때마다 새 버전 생성, 이전 버전 보존
- 대본 승인 후 구조화 진행

### 3.2 대본 구조화 (Scene → Shot → Frame)

- Scene: 논리적 장면 단위. 배경(setting), 분위기(mood), 예상 길이
- Shot: 카메라 샷 단위. 샷 타입(wide/medium/close-up), 카메라 움직임, 길이
- FrameSpec: 생성 가능한 최소 단위. visual prompt, negative prompt, dialogue, duration, transition
- 각 레벨에서 수동 편집 가능 (순서 변경, 추가, 삭제)

### 3.3 캐릭터·스타일 일관성

- **CharacterProfile**: 이름, 외형 설명, visual prompt 조각, 참조 이미지, 음성 ID
  - 해당 캐릭터가 등장하는 모든 FrameSpec에 자동 주입
- **StylePreset**: 프로젝트 전역 스타일 가이드
  - prompt prefix/suffix, negative prompt, 선호 모델 설정
  - "시네마틱", "애니메이션", "다큐멘터리" 등 사전 정의 가능

### 3.4 에셋 생성

| 에셋 타입 | 프로바이더 | 입력 | 출력 |
|-----------|-----------|------|------|
| 이미지 | fal.ai (Flux 등) | visual prompt + style + character | PNG/WebP |
| 비디오 | Runway Gen-3 | 이미지 + motion prompt | MP4 클립 |
| TTS 음성 | ElevenLabs | dialogue + voice_id | WAV/MP3 |
| 자막 | 내부 (TTS 타이밍 기반) | VoiceTrack timestamps | SRT/ASS |

### 3.5 타임라인 & 렌더링

- Timeline: Scene/Shot/FrameSpec 순서에 따른 세그먼트 자동 배치
- 트랜지션: cut, fade, dissolve (FrameSpec.transition_type)
- 오디오 레이어: TTS 음성 + BGM (v2)
- 자막 오버레이
- FFmpeg 기반 최종 렌더 → mp4 출력

### 3.6 작업 관리

- 비동기 Job Queue (arq + Redis)
- 진행률 추적 (0–100%)
- 실패 Job 개별 재시도
- ProviderRun으로 모든 외부 API 호출 로깅 (파라미터, 응답, 지연시간, 비용 추정)

## 4. v1 / v2 범위

### v1 — 견고한 생성 파이프라인

**포함:**
- 프로젝트 CRUD + ProjectSettings
- AI 대본 생성 + ScriptVersion 관리
- Scene/Shot/FrameSpec 자동 구조화 + 수동 편집
- CharacterProfile + StylePreset
- 이미지·비디오·TTS 생성 (fal.ai, Runway, ElevenLabs)
- 자막 자동 생성 (TTS 타이밍 기반)
- Job Queue + 부분 재생성
- Timeline 합성 + FFmpeg 렌더
- ProviderRun 로깅
- 기본 웹 UI (프로젝트 관리, 대본 편집, 생성 컨트롤, 프리뷰)

### v2 — 사용성 고도화

- BGM 자동 생성/선택
- 영상 트리밍·컷 편집
- 프롬프트 히스토리·즐겨찾기
- 배치 생성 (여러 프로젝트 병렬)
- A/B 비교 (동일 FrameSpec, 다른 설정)
- 커스텀 StylePreset 에디터
- 고급 타임라인 편집 (구간 재배치, 오디오 믹싱)

## 5. 고품질 영상 제작을 위한 핵심 원칙

### 원칙 1: 프레임 단위 제어가 핵심이다

Scene 수준의 프롬프트만으로는 시각적 일관성을 확보할 수 없다.
각 FrameSpec이 독립적인 visual prompt를 갖고, 사용자가 직접 조정할 수 있어야 한다.

### 원칙 2: 캐릭터 일관성은 시스템이 강제해야 한다

CharacterProfile의 visual prompt + 참조 이미지가 **자동으로** 모든 관련 생성 요청에 주입된다.
사용자가 매번 수동 입력하면 실수가 생기고 일관성이 깨진다.

### 원칙 3: 스타일은 프로젝트 전역에서 일관되어야 한다

StylePreset에 정의된 prompt prefix/suffix/negative가 모든 visual prompt에 자동 append된다.
프레임마다 다른 스타일이 적용되면 영상 전체의 톤이 무너진다.

### 원칙 4: 실패는 부분 재시도로 해결한다

전체 재생성은 시간과 비용의 낭비다.
FrameSpec 1장, Shot 1개, Scene 1개 단위로 재생성이 가능해야 한다.
실패한 Job만 재시도하는 것이 기본 동작이다.

### 원칙 5: 모든 생성은 추적 가능해야 한다

어떤 프롬프트로, 어떤 모델에, 언제 요청했고, 응답은 무엇이었는지.
ProviderRun에 기록하여 디버깅·비용 추적·품질 개선에 활용한다.

## 6. 로그인/멀티유저를 제외하는 이유

| 이유 | 설명 |
|------|------|
| 1인 내부 도구 | 제작자 본인만 사용. 계정 분리 자체가 불필요 |
| 복잡도 최소화 | 인증·권한 레이어 없이 생성 파이프라인 로직에 온전히 집중 |
| 로컬/개인 서버 배포 | localhost 또는 VPN 접근만 허용. 네트워크 레벨 보안으로 충분 |
| 빠른 반복 | 세션 관리·토큰 갱신 등 부수 작업 없이 즉시 개발 가능 |
| 추후 확장 가능 | 필요 시 auth 미들웨어 추가 + user_id 컬럼 확장으로 전환 가능 |

인증이 없다고 해서 보안을 무시하는 것은 아니다.
API 키 등 민감 정보는 환경변수로 관리하고, 서비스는 외부에 노출하지 않는다.
